## 📌 문제 개요
체크된 공부 날짜들이 주어질 때, 결석한 날을 최대 `p`번까지 채워 넣어 만들 수 있는 **가장 긴 연속 공부 기간**을 구하는 문제입니다. 투 포인터를 활용하여 탐색 범위를 최적화하고 시간 복잡도를 줄이는 것이 핵심입니다.

---

## 💡 풀이 핵심 (Core Logic)

### 1. [미래 예측 전략](./solution.c++)
*   right 포인터를 기준으로 '다음 날짜(right + 1)'를 보며 빈칸을 채울 수 있는지 확인합니다.
*   **장점:** O(N)의 시간 복잡도로 전체 공부 기간의 범위에 구애받지 않고 빠르게 탐색합니다.
*   **단점:** while문 내부에서 right + 1을 참조하다 보니 배열의 끝(n-1)을 처리하기 위한 분기 조건이 추가되어 코드가 다소 복잡해집니다.

### 2. [현재 수습 전략](solution_ver2.c++)
*   right 포인터는 단순 전진만 시키고, 문제가 생기면 left 포인터를 당겨서 '현재를 수습'하는 방식을 사용합니다.
*   **특징:**  로직의 주도권을 for문(right 확장)과 while문(left 수습)으로 명확히 분리하여 성능 저하 없이 가독성을 크게 끌어올렸습니다.
*   **포인트"** 최소 보장 일수인 p + 1을 res의 초기값으로 설정하여 불필요한 예외 처리를 걷어냈습니다.

### 3. [번외 : Undefined Behavior 풀이](solution_ver3.c++)
*   코드의 일관성을 위해 right를 0부터 시작하게 만든 코드입니다.
*   **특징:** 루프 마지막에 right++가 실행되어 right가 n이 되면, 바로 다음 줄에서 date[n]을 참조하게 되므로, 자바(Java) 였다면 **ArrayIndexOutOfBoundsException**을 뱉으며 실행이 불가능한 코드입니다.
*   **c++은 가능한 이유:** c++의 메모리 관리 차이 때문입니다. 배열 범위를 벗어나 메모리 어딘가 '쓰레기 값'을 참조하여 p가 엉뚱하게 계산되었지만, 그 값을 가지고 최댓값을 갱신하기 직전에 while(right < n) 루프 조건에 걸려 프로그램이 종료되었기 때문입니다.

## 📊 풀이 방식 비교 (미래 예측 vs 현재 수습)

| 비교 항목 | 풀이 1: 미래 예측형 (`while` 중심) | 풀이 2: 현재 수습형 (`for-while` 중심) |
| :--- | :--- | :--- |
| **관찰 시점 (Focus)** | `right + 1` (다음 칸의 상태를 미리 확인) | `left` (현재 누적된 상태를 확인) |
| **핵심 흐름** | "다음 칸으로 가도 안전한가?" 확인 후 `right` 이동 | 일단 `right` 전진, 한도 초과 시 `left` 당겨서 수습 |
| **루프 구조** | `while`문 내부에 `while`문이 중첩된 형태 | `for`문으로 전진하고 `while`문으로 제어하는 형태 |
| **인덱스 안전성** | `right < n - 1` 등 범위 초과를 수동으로 방어해야 함 | `for (right < n)` 조건이 경계를 알아서 안전하게 보장함 |
| **특수 분기 처리** | `if (left == right)` 등 예외 상황에 대한 수동 `if`문 필요 | 별도 `if`문 없이 조건(`p < 0`)에 의해서만 자연스럽게 동작 |
| **가독성 및 유지보수** | 인덱스 제어와 계산 로직이 얽혀 있어 뇌 버퍼가 쉽게 참 | **확장 → 수습 → 기록**의 단방향 흐름으로 논리가 매우 투명함 |
